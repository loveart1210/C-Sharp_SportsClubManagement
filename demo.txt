1. Công nghệ Front-end đề xuất
Dùng C# Desktop Application – Offline Windows App
WinForms (.NET Framework) ưu tiên đơn giản

2. Triết lý thiết kế UI tổng thể
•	Role-based UI: Hiển thị chức năng theo quyền
•	Desktop-first: Chuột + bàn phím
•	Offline-friendly: Không phụ thuộc internet
•	Phân tách rõ module: Team / User / Lịch / Quỹ / Thông báo
•	Nhất quán layout: Sidebar + Main Content
•	KHÔNG SỬ DỤNG KÉO THẢ
•	Tất cả các trang đều có nút “quay lại” để quay lại trang vừa truy cập

3. Layout tổng thể ứng dụng
3.1 Cấu trúc màn hình chính (Main Window)
+------------------------------------------------+
|  Top Bar                                      |
|  - App Name                                   |
|  - User Avatar + Name + Role                  |
|  - Logout                                     |
+------------------------------------------------+
| Sidebar          | Main Content                |
|------------------|----------------------------|
| - Dashboard      |                            |
| - Profile        |                            |
| - Teams          |  Nội dung module           |
| - Calendar       |                            |
| - Notifications  |                            |
| - Admin Panel*   |                            |
+------------------------------------------------+
* Admin Panel chỉ hiện nếu role = Admin hệ thống
3.2 Sidebar Navigation (Left Menu)
Menu items theo role:
Chung (Admin + User):
•	Dashboard
•	Trang cá nhân
•	Team của tôi
•	Lịch tập cá nhân
Admin hệ thống:
•	Quản lý User
•	Quản lý toàn bộ Team

5. Module Authentication (Đăng nhập)
Màn hình Login:
Components
•	TextBox: Username
•	PasswordBox: Password
•	Button: Login
•	Label: Error message
Yêu cầu UI
•	Không lưu password
•	Không hiện password
•	Thông báo lỗi rõ ràng

6. Module Trang cá nhân (Profile)
6.1 Layout
[Avatar]  [Change Avatar]

Họ tên:     [TextBox]
Ngày sinh:  [DatePicker]
Email:      [TextBox]
Username:   [TextBox] (Read-only)

[Save]   [Cancel]   [Change Password]
6.2 Chức năng
•	CRUD avatar
•	CRUD họ tên, ngày sinh, email, username
•   Change Password (User reset own)
•	Validate:
o	Email format
o	Ngày sinh không vượt hiện tại

6.5 Module Lịch tập cá nhân
Chức năng:
•   Quản lý môn tập cá nhân (CRUD dạng lịch)
•   Quản lý lịch tập cá nhân (CRUD dạng lịch)
UI:
•   Tương tự Module Calender nhưng áp dụng cho cá nhân.
